需要实现：
    漫画NFT铸造
    核心市场功能：创造者变现、版税变现
    跨链集成
    

构建：
    阶段一：基础数据结构、错误定义、处理计算的库（不依赖任何合约）
    │
    ├── DataTypes.sol       先定义所有数据结构
    ├── Errors.sol          定义所有自定义错误定义
    └── PercentageMath.sol  实现版税计算，写单元测试验证数字

    阶段二：NFT核心（只依赖地基）
    │
    ├── IComicNFT.sol       接口明确对外暴露什么：
    └── ComicNFT.sol        实现铸造、元数据存储、IP家族、IP授权以及审核
    └── 测试：mint、衍生品注册、版税配置读取
    实现：
        ERC721（代币标准）
        ERC-2981（版税标准）
            通过实现 royaltyInfo 函数和 supportsInterface ，该NFT合约可以被 OpenSea 等主流第三方市场正确识别版税信息
            royaltyInfo: 用户在市场上转售NFT时，获取版税金额 → 自动分给创作者，剩余的转给卖家
                      ↑
                市场合约调用

    阶段三：版税拆分（依赖ComicNFT）
    │
    ├── IRoyaltySplitter.sol
    ├── RoyaltySplitter.sol  实现 splitProceeds + pull模式提取： 实现 splitProceeds + pull模式提取：先分配到待提取账户，实际转账由各方主动提取（pull 模式，防止 push 失败导致交易回滚）
    │      distribute、withdraw
    └── 测试：各种比例计算、withdraw

    阶段四：市场交易（依赖前两阶段，最复杂）
    │
    ├── IMarketplace.sol
    ├── Marketplace.sol      挂单→购买→版税拆分完整流程
    │        固定价格挂单 ( list , cancelListing ) ：实现了最基础的NFT上架和下架功能。
    │        购买 ( buy , batchBuy ) ：支持单个和批量购买，并通过内部的 _executeBuy 函数统一处理核心逻辑。
    │        出价、接受出价 ( makeOffer , acceptOffer ) ：提供了点对点的议价机制。
    │        英式拍卖 ( createAuction , bid , settleAuction ) ：实现了一套完整的拍卖流程，包括起拍价、保留价、加价逻辑和结算。  
    └── 测试：买卖、版税自动分配、退款、暂停

    阶段五：扩展模块（相对独立）
    │
    ├── BountyBoard.sol      悬赏全流程+超时保护
    └── 测试：完整悬赏生命周期

    阶段六：代理层（功能稳定后才加）
    │
    ├── 五个 Proxy 合约
    ├── Deploy.s.sol
    └── Upgrade.s.sol

    阶段七：跨链（最后，依赖外部协议）
    │
    └── CrossChainBridge.sol  
            去程 ：A链 bridgeOut (锁定) -> CCIP -> B链 ccipReceive (铸造)。bridgeOut：锁定原始NFT，发起跨链；ccipReceive：在目标链铸造封装 NFT
            回程 ：B链 bridgeIn (销毁) -> CCIP -> A链 ccipReceive (解锁)。bridgeIn：销毁目标链上的封装 NFT。ccipReceive：在目标链铸造封装 NFT，解锁原始 NFT
            


阅读代码顺序建议：
    第一步：看文档（10分钟）
        看 README.md 和 开发文档.md、需求文档.md。搞清楚这个项目是什么、有哪些角色、钱怎么流动。脑子里先有业务图，再看代码不会迷失。

    第二步：看 libraries/（30分钟）
        从最底层开始，顺序是：
            DataTypes.sol       ← 先看，所有数据结构都在这里
            Errors.sol          ← 快速过，知道有哪些错误类型
            PercentageMath.sol  ← 看懂 splitProceeds()，这是版税计算的核心
        这三个文件看完，你就知道整个系统的"数据语言"是什么，后面看合约不会陌生。

    第三步：看 interfaces/IAll.sol（15分钟）
        不看实现，只看接口。每个接口定义了这个合约"能做什么"、"会触发什么事件"。相当于看目录，建立整体印象。
    
    第四步：按依赖顺序看核心合约
        顺序很重要，因为后面的合约依赖前面的：
            1. ComicNFT.sol        ← 铸造、IP家族、授权，是一切的起点
            2. RoyaltySplitter.sol ← 钱怎么分，逻辑相对独立
            3. Marketplace.sol     ← 最复杂，调用前两个
            4. BountyBoard.sol     ← 独立模块，最后看
            5. CrossChainBridge.sol← 最后看，依赖外部协议
         每个合约重点看三个位置：initialize()（初始状态）、核心业务函数、事件定义。
    
    第五步：看测试文件
        test/ComicMarket.t.sol 是最好的"使用说明书"，每个 test_ 函数都是一个完整的业务流程。比文档更准确，因为能跑通。
    
    第六步：看代理和部署
        最后看 proxy/ 和 script/Deploy.s.sol，理解合约怎么部署、怎么升级。
